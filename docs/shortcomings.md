# 项目短板与优先补齐项

> 状态快照：本地 DXF 批量翻译可用；真 DWG 闭环与产品化未完成。  
> 仓库：https://github.com/erict16/dwg-converter-pro

---

## 1. 原生 DWG 还没真正打通（最大）

- 本机若未装 **ODA File Converter**，链路只完整验证了 **DXF**。
- 业务样例几乎都是 **.dwg**，不装 ODA 只能先手工转 DXF。
- 即使装了 ODA：转换稳定性、CAD 版本兼容、块/注释写回仍可能有坑。
- **真实大图**（`CAD\中文图纸` 等）尚未做过系统回归。

**影响：** 目标用户拖 DWG 进来可能直接失败。

---

## 2. 翻译质量与实体覆盖不完整

| 点 | 现状 |
|----|------|
| 术语表 | 约 393 条，最长匹配；新词、整句、口语化标注仍漏 |
| 机翻兜底 | 默认 MyMemory：不稳定、有限额、电力术语易翻错 |
| MULTILEADER 引线 | best-effort；不同 CAD 版本结构不一，可能漏箭头注释 |
| 表格等 | `TABLE` 等基本未处理 |
| 混排与格式 | MTEXT 格式码部分保留，复杂格式可能乱 |

术语表命中时质量高（如「切换开关」→ diverter switch）；**表外句子**依赖通用机翻，风险大。

---

## 3. 架构仍是「本机双进程」，不是成品 SaaS

- 前端可上 **Vercel**，CAD Worker **不能**放进 Vercel Serverless。
- 本地需 **两个终端**（web + worker），普通用户门槛高。
- 无登录、无配额、无任务中心（仅简单状态）。
- 浏览器直连 `127.0.0.1:8000`；上云需 CORS、HTTPS、鉴权与公网 Worker。

---

## 4. 体验与产品层偏薄

- 不能在网页 **预览** 译前/译后图纸（需下载用 CAD 打开）。
- 没有「译文人工修改后再出图」。
- 错误提示对 DXF/ODA 有说明，可操作性仍一般。
- 深浅色主题已有，整体仍是开发者向工具站。

---

## 5. 测试与验收不够「真图」

- 现有测试以 **合成 DXF + pytest** 为主。
- 未用 `CAD\中文图纸` 四张真图做固定验收。
- 未与 `CAD\SHZV` 英文参考图做用词/版式对照。
- 无大文件、多文件并发压测。

---

## 6. 运维与依赖

- Node 可能依赖本机便携路径，换机器要改脚本或 PATH。
- Worker 临时目录有清理（保留最近任务），异常中断时的清理仍简单。
- 公开仓库含术语表：对公司词表公开是否合适需自行把关。

---

## 核心三句话

1. **DWG + ODA 真图闭环未完成**  
2. **术语外翻译 / 引线等实体覆盖不可靠**  
3. **仍是开发者向本地工具，不是一键可用的线上产品**

---

## 建议优先补齐顺序

| 优先级 | 事项 | 预期收益 |
|--------|------|----------|
| P0 | 安装 ODA，用中文图纸真 DWG 跑通并修 bug | 对上真实使用场景 |
| P0 | 失败时输出「未译中文列表」 | 质量可审计 |
| P1 | 加强 MULTILEADER / 块属性覆盖 | 箭头注释等少漏 |
| P1 | 对照 SHZV 英文图 spot-check 用词 | 专业用语一致 |
| P2 | 译文预览 / 人工改后再生成 | 可用性与安全 |
| P2 | Worker 部署方案（本机常驻或云主机）+ 前端对接 | 接近产品化 |
| P3 | 队列、鉴权、预览缩略图 | 完整产品 |

---

## 当前已具备的能力（对照）

- 多文件拖拽队列 + 批量翻译 + ZIP 下载  
- 黑/白主题  
- 术语表最长匹配 + 可选机翻兜底  
- DXF 文本实体改写（TEXT / MTEXT / ATTRIB / 部分引线与尺寸）  
- pytest 与本地 DXF 实译验证通过
